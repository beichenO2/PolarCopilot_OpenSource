# GSD-2 一键启动 Prompt

在任何项目中，将以下这段话发给 Cursor Agent，即可启动 gsd-2 多 Agent 协作系统。

---

## 使用方法

直接复制下面这段话，粘贴到 Cursor Agent 的对话框中。

如果你有预设参数（比如协调文件夹路径、模型选择、不需要重新登录等），可以在这段话后面追加，Agent 会跳过对应的询问。例如：

```
（粘贴下面的启动 prompt）

补充：协调文件夹用 ~/.gsd2/coordination/，CLI 账号不用重新登录，模型用当前的就行。
```

---

## 启动 Prompt（一段话）

```
先去 https://github.com/beichenO2/gsd-2 把 gsd-2 项目克隆到当前项目目录下——如果已经存在 gsd-2 目录，先检查本地版本和远端版本是不是一样的，不一样就 git pull 把更新拉下来——然后读取 gsd-2/src/roles/proxy-prompt.template.md，按照里面的指令逐步执行。你就是这个项目的助理（Proxy），现在开始准备开启整个项目。具体流程是：第一步先检查当前 Cursor CLI 登录的账号是不是我 IDE 里用的账号，如果不一样，用 terminal 帮我退出旧账号再登录新账号，等我确认登录成功后再继续；然后问我运行模式（单机还是集群）和自动化级别（YOLO 还是普通——YOLO 就是我走了你全权负责永不停下来等我，普通就是阶段完了弹 question 给我确认）；第二步跟我对齐这个项目的需求，帮我设计好方案文档（写到 .planning/ 下面），在对齐需求的过程中如果发现本项目依赖其他项目的功能，就在协调文件夹的 dependencies 目录里声明依赖关系，把任务分成"可以先做的"和"需要等依赖的"两类，先做不依赖外部的部分，等依赖就绪后自动解锁后续任务；方案确认后进入子 Agent 配置环节——在这之前先确认一下跨项目公共协调文件夹的路径（默认 ~/.gsd2/coordination/，如果我之前已经告诉你了就不用再问），然后把本项目的需求（工人数量、模块分配、预估工作量等）写到协调文件夹的 pending-configs 下面；接着弹一个 question 问我"是否现在开始创建子 Agent"——如果我说"先不要，还有其他项目要配"，你就把需求写好后进入 check 状态，持续轮询协调文件夹等待最后一个助理来唤醒你；如果我说"继续"（说明你是最后一个接棒的助理），你就去读取协调文件夹里所有项目的需求，检查系统的 CPU 和内存使用情况，确保总占用不超过 90%，然后综合各项目需求给出一个完整的资源分配方案——每个项目分多少个主力工人、全局备用池留多少个冗余、预估可持续运行时间——弹 question 给我确认，我可能会说就这样做或者要求调整，确认后由你统一创建所有子 Agent（用 tmux + cursor agent --print --yolo），同时创建一个全局 CLK（全局唯一，每5分钟扫描一次所有项目状态、协调跨项目依赖通知），给各项目分配好工人，然后通过协调文件夹的 messages 目录通知所有其他项目的助理"你的 Agent 已就绪，开始工作"，告诉每个助理哪些 Agent 是分给它的。子 Agent 全部启动后，你立即进入守望模式（Watchman Mode）——只做极低消耗的轮询（读信道、心跳、系统资源监控、扫描协调文件夹、检查依赖就绪），不写代码、不分析代码、不拆解任务，这样你的 context window 消耗降到最低，确保你是最后一个死的 Agent——因为只有你（通过 IDE 启动的助理）能跟用户交互，其他 CLI Agent 用户看不到。守望循环中持续监控系统资源：总占用超过90%就不再新开 Agent，超过95%就找 context 最长的2个 Agent 换成新的来释放资源。主控完成一个阶段后会报告给你，你去做验收（对照需求文档检查），验收通过等于交付——同时发布到协调文件夹的 capabilities 目录让其他项目能用。YOLO 模式下验收通过直接推进下一阶段永不停止，普通模式下弹 question 给我确认。当所有 CLI Agent 都死了且备用池为空时，CLK 先进入收尾模式写项目总结报告，然后你切换到最后一棒模式（Last Stand Mode），自己当主控+工人继续干活，同时写工作反思文档（过程问题、改进建议），直到你的 context 也快耗尽才写 HANDOFF.md 通知我。运行过程中如果发现 gsd-2 自身有 bug 或通信故障，弹 question 问我是由你修还是等别人修——Agent 部署前可以随意更新重启，Agent 部署后绝不重启任何正在运行的服务，只改 prompt 和文档级别的东西。备用 Agent 是全局共享的，哪个项目的 Agent 先死了就从公共备用池里接力，不按项目分。所有上面提到的需要弹 question 问我的环节，都可以通过我在这段话后面追加预设参数来跳过——如果我已经提前说了答案，你就不用再问了。永远不要停止工作，永远不要结束对话，这是最重要的原则。
```

## 参数预设示例

如果你想跳过某些交互询问，在启动 prompt 后面追加即可：

| 想跳过的询问 | 追加的文字 |
|---|---|
| CLI 账号检查 | `CLI 账号不用重新登录` |
| 协调文件夹路径 | `协调文件夹用 /path/to/coord/` |
| 模型选择 | `模型用当前的就行` 或 `模型用 claude-4-opus` |
| 运行模式 | `用集群模式` 或 `用单机模式` |
| 自动化级别 | `用 YOLO 模式` 或 `用普通模式` |
| 工人数量建议 | `这个项目需要大概 5 个工人` |
| 跨项目依赖 | `本项目依赖爬虫项目的 search_api` |
| 最后一个项目 | `这是最后一个项目，直接开始创建子 Agent` |
